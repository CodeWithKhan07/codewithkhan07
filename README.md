### About

I build production Flutter apps end-to-end — architecture, implementation, and shipping to the App Store. My work centers on **Riverpod**, clean architecture, and SOLID principles, currently applied to a server-driven UI (SDUI) system at scale.

At **Chimpgroup**, I work on **QDrive**, a vehicle rental platform where entire screens are defined as JSON on the backend and rendered dynamically on the client — a "ship once" approach that removes the need for App Store review on most UI changes. Before that, at **EmergeSol Solutions**, I was the sole Flutter developer on three shipped production apps, and independently built a browser-extension suite automating listings across European marketplaces on a shared Firebase backend.

I'm equally comfortable across state management approaches (Riverpod, GetX, BLoC), backend integration (Firebase, REST), and offline-first design (Hive, Sqflite) — choosing based on what a codebase actually needs rather than defaulting to one tool.
